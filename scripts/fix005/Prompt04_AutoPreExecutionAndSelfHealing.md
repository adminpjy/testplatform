# Prompt04：自动预执行与自愈验证

## 目标

对 LLM 生成的修复建议进行自动预执行验证，验证通过后再允许应用。

## 自愈流程

1. 失败发生。
2. 生成 evidence package。
3. LLM 诊断。
4. 生成规则草案/DSL patch/测试数据 patch。
5. dry-run。
6. 预执行验证。
7. 低风险自动应用或提示用户一键应用。
8. 高风险进入人工审核。
9. 验证失败则生成反馈。

## 预执行限制

1. 不执行真实删除。
2. 不执行真实作废。
3. 不执行真实审批提交，除非测试账号、用例和风险规则均明确允许。
4. 预执行默认停在最终提交前。

## 验收标准

1. 定位类规则能自动验证。
2. 表单字段规则能用失败页面验证。
3. 审批类规则只验证意见框和按钮识别，不默认提交。
