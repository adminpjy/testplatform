# Prompt00：数据库备份与清空重建策略

## 背景补充

fix005 的重点是 LLM 自愈闭环。历史失败样本、旧分析记录、旧人工介入记录不需要兼容。  
如需让证据包、修复建议、规则草案、预执行记录形成更清晰的数据模型，可以重建结构。

## 执行前必须做

1. 备份当前数据库到 `backups/database/YYYYMMDD-HHMMSS/`。
2. 备份 `data/aitp.db`，如存在也备份 `data/aitp.db-wal` 和 `data/aitp.db-shm`。
3. 确认备份文件存在。

## 允许

1. 清空历史失败样本。
2. 清空历史人工介入记录。
3. 清空历史 LLM 分析记录。
4. 重建自愈闭环相关数据结构。
5. 调整 evidence package 的存储结构。

## 原则

1. 自愈链路清晰优先。
2. 证据完整优先。
3. 用户可理解优先。
4. 不为兼容旧数据牺牲新流程。
