# Prompt04：规则库、页面知识库、Prompt 配置低代码化

## 目标

降低维护门槛，让非专业维护人员也能配置规则、页面知识和 Prompt。JSON 只保留为高级模式，不作为主要操作方式。

## 规则库设计

### 简单模式

表单字段：

1. 规则名称。
2. 规则类型。
3. 适用项目/系统/页面/用例。
4. 业务目标。
5. 页面特征。
6. 目标控件。
7. 操作方式。
8. 成功标准。
9. 风险等级。
10. 是否需要人工确认。

### 向导模式

按规则类型提供向导：

1. 登录规则向导。
2. 菜单导航规则向导。
3. 列表规则向导。
4. 查询规则向导。
5. 表单规则向导。
6. 审批规则向导。
7. 弹窗规则向导。
8. 断言规则向导。

### 高级模式

保留 JSON 编辑器，但必须：

1. 有字段说明。
2. 有示例。
3. 有 Schema 校验。
4. 有 LLM 解释。
5. 有风险检查。

## 页面知识库设计

从 JSON 展示升级为业务卡片：

1. 页面名称。
2. 页面类型。
3. URL 模式。
4. 菜单路径。
5. 关键表格。
6. 关键表单。
7. 关键按钮。
8. 审批区域。
9. 成功证据。
10. 关联规则。
11. 关联失败样本。
12. 最近命中次数。

## Prompt 配置设计

Prompt 维护页面需要支持：

1. 场景化分类。
2. 版本管理。
3. 变量说明。
4. 示例输入。
5. 试运行。
6. 新旧版本对比。
7. 回滚。
8. LLM 辅助优化。

Prompt 场景至少包括：

1. 自然语言解析。
2. DSL 生成。
3. 失败分析。
4. 规则生成。
5. 人工介入方案。
6. 测试数据生成。
7. 报告总结。

## LLM 辅助功能

每个配置页面都应提供：

1. 解释当前配置。
2. 根据描述生成配置。
3. 根据失败样本生成配置。
4. 检查配置问题。
5. 生成验证计划。
6. 生成示例。

## 验收标准

1. 不写 JSON 也能创建常见规则。
2. 非专业人员能看懂页面知识。
3. Prompt 能试运行并比较结果。
4. LLM 能辅助生成和检查配置。
