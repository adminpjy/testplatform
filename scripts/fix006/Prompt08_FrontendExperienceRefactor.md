# Prompt08：前端体验重构

## 目标

让非专业维护人员能使用系统完成维护升级，不需要理解复杂 JSON 和底层执行细节。

## 通用体验

1. 所有列表页统一分页。
2. 所有列表页有筛选区。
3. 重要状态用中文可读标签。
4. 每个页面提供主操作按钮。
5. 每个错误有“下一步建议”。
6. JSON 编辑隐藏到高级模式。

## 页面改造

### 规则库

1. 列表分页。
2. 规则类型筛选。
3. 作用域筛选。
4. 状态筛选。
5. 表单化创建。
6. 向导化创建。
7. LLM 辅助生成。
8. JSON 高级模式。

### 页面知识库

1. 分页。
2. 页面类型筛选。
3. 项目筛选。
4. 业务卡片展示。
5. 关联规则和失败样本。

### 失败样本

1. 升级为失败处理工作台。
2. 列表分页。
3. 一键分析。
4. 一键生成方案。
5. 一键反馈。

### 人工介入

1. 列表分页。
2. 查看介入前后证据。
3. 一键总结为规则。
4. 一键验证规则。

### Prompt 配置

1. 场景分类。
2. 变量说明。
3. 试运行。
4. 版本对比。
5. LLM 优化。
6. 高级 JSON/模板模式。

### 测试报告

1. 项目总览。
2. 模块下钻。
3. 用例下钻。
4. 运行下钻。
5. 步骤证据查看。

## 验收标准

1. 非专业人员可以通过页面完成常见规则创建。
2. 失败样本可从“看到问题”走到“生成方案/反馈”。
3. 报告可按项目查看总览并下钻。
4. 前端构建通过。
