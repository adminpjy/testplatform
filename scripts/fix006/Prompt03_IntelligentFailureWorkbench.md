# Prompt03：失败样本智能工作台

## 目标

失败样本不再只是展示记录，而是待处理问题队列。用户进入失败样本页面后，可以看到原因、方案、风险、验证结果，并一键处理。

## 页面定位

将“失败样本”升级为“失败处理工作台”。

## 列表字段

1. 失败摘要。
2. 项目。
3. 用例。
4. 运行记录。
5. 失败步骤。
6. 失败分类。
7. 风险等级。
8. 分析状态。
9. 修复状态。
10. 验证状态。
11. 创建时间。
12. 操作。

## 详情页内容

1. 当前问题说明。
2. 截图证据。
3. 过程截图。
4. 失败步骤详情。
5. 页面上下文。
6. 命中规则。
7. LLM 分析结果。
8. 推荐修复方案。
9. 可执行动作。

## 智能动作

必须提供：

1. `智能分析`
   - 调用 LLM 分析 evidence package。

2. `生成解决方案`
   - 输出规则草案、DSL patch、测试数据 patch、人工介入方案。

3. `生成规则草案`
   - 从失败样本生成可审核规则。

4. `预执行验证`
   - dry-run 或基于失败现场验证。

5. `一键应用`
   - 低风险方案允许一键应用。
   - 高风险方案必须人工确认。

6. `重跑验证`
   - 对用例重新执行，验证修复有效。

7. `标记为系统缺陷`
   - 生成缺陷候选。

8. `一键反馈`
   - 打包完整证据给维护后台。

## 用户可读要求

不要直接展示 Python 异常作为主信息。  
主信息必须是：

1. 系统想做什么。
2. 当前页面是什么状态。
3. 为什么失败。
4. 可以怎么处理。
5. 哪个方案最推荐。

## 验收标准

1. 失败样本列表支持分页、筛选。
2. 每个失败样本可智能分析。
3. 分析后可生成解决方案。
4. 可一键生成规则草案。
5. 可一键反馈。
