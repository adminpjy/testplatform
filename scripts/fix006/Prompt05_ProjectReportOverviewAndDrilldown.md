# Prompt05：项目级报告总览与下钻

## 目标

测试报告从“单次运行报告”升级为“项目级测试质量报告”，支持从项目总览下钻到用例、运行、步骤和证据。

## 报告层级

1. 项目总览。
2. 测试活动/批次。
3. 模块。
4. 功能点。
5. 用例。
6. 运行记录。
7. 步骤。
8. 证据。

## 项目总览指标

1. 项目名称。
2. 被测系统。
3. 测试范围。
4. 用例总数。
5. 已执行用例数。
6. 未执行用例数。
7. 通过数。
8. 失败数。
9. 阻塞数。
10. 通过率。
11. 失败率。
12. 最近一次批次结果。
13. 趋势图。
14. 模块覆盖。
15. 正例/反例/边界覆盖。
16. 高风险操作统计。
17. 自动修复次数。
18. 人工介入次数。
19. 未解决问题数。

## 下钻

### 项目 -> 模块

展示模块通过率、失败数、未执行数。

### 模块 -> 用例

展示用例状态、最近运行、失败原因、责任建议。

### 用例 -> 运行

展示历史运行趋势和版本变化。

### 运行 -> 步骤

展示每一步执行详情、截图、过程截图、日志。

## 导出

支持：

1. HTML。
2. Word/PDF。
3. Excel 明细。

第一期至少实现 HTML 总览和 Excel 明细设计接口。

## LLM 报告总结

LLM 生成：

1. 总体质量结论。
2. 主要风险。
3. 高频失败原因。
4. 建议优先修复项。
5. 是否具备上线建议。

## 验收标准

1. 能按项目查看总览报告。
2. 能下钻到某一用例。
3. 能查看用例运行和步骤证据。
4. 能看到失败分类和建议。
