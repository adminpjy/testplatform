# Prompt06：维护助手与 LLM 辅助流程

## 目标

让非专业维护人员可以通过自然语言完成维护任务，例如：

1. “帮我分析这类失败。”
2. “帮我生成适配这个页面的规则。”
3. “帮我检查这个 Prompt 有什么问题。”
4. “帮我把人工介入步骤沉淀成规则。”
5. “帮我总结这个项目的测试质量。”

## 维护助手入口

在以下位置提供 LLM 辅助：

1. 失败样本详情。
2. 人工介入详情。
3. 规则编辑页。
4. 页面知识详情。
5. Prompt 配置页。
6. 项目报告页。

## LLM 输入

根据场景自动整理上下文，不要求用户复制粘贴：

1. 失败证据包。
2. 当前规则。
3. 页面知识。
4. Prompt 内容。
5. 用例和 DSL。
6. 截图摘要。
7. 运行日志。

## LLM 输出

必须结构化：

1. 用户可读说明。
2. 建议动作。
3. 风险等级。
4. 影响范围。
5. 可执行 patch。
6. 验证计划。
7. 是否需要人工审核。

## 一键动作

LLM 输出后可一键：

1. 创建规则草案。
2. 修改用例草案。
3. 修改 Prompt 草案。
4. 创建人工介入方案。
5. 创建反馈工单。
6. 创建缺陷候选。
7. 启动验证。

## 安全要求

1. LLM 不直接修改生产启用配置。
2. LLM 建议必须进入草案或待确认状态。
3. 所有 LLM 输入输出记录审计。
4. 敏感信息必须脱敏。

## 验收标准

1. 维护人员不用理解底层 JSON 也能生成配置草案。
2. LLM 建议可解释、可验证、可回滚。
3. 高风险建议不会被自动执行。
