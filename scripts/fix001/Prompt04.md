当前项目是“企业 MIS 智能功能测试平台”，不是 NISF，不做代码生成。

现在开始第三阶段：实现前端项目、测试账号、功能测试用例管理页面。

目标：
用户不再每次临时输入测试目标，而是在项目下维护可重复执行的功能测试用例。

============================================================
一、导航调整
============================================================

前端主导航应包含：

- 项目管理
- 测试运行
- 能力中心
- 失败样本
- 测试报告
- 系统设置

“被测系统管理”可以合并到“项目管理”中。
前端概念统一为：
项目（被测系统）

============================================================
二、项目管理页
============================================================

路径建议：
/projects

列表展示：
- 项目名称
- 被测系统名称
- base_url
- 默认账号
- 用例数量
- 最近运行状态
- 状态
- 操作

操作：
- 新增项目
- 编辑项目
- 删除 / 停用项目
- 进入项目详情
- 连通性检查
- 登录检查

新增/编辑项目弹窗字段：
- 项目名称
- 描述
- 被测系统名称
- base_url
- login_url
- home_url
- auth_type
- 默认执行设置：
  - Trace
  - Screenshot
  - DOM Snapshot
  - Accessibility Snapshot
  - Vision Fallback

要求：
1. 不再报 ALLOWED_BASE_URL_PREFIXES。
2. 没有项目时提示“请先创建项目”。
3. 不再自动创建隐藏默认项目。
4. 已有默认测试项目也应显示出来，可编辑、停用或删除。

============================================================
八、DSL 设计 Tab
============================================================

功能：
- 可视化步骤列表；
- DSL JSON 编辑器；
- 格式化按钮；
- 校验按钮；
- 保存为新版本；
- 重新生成 DSL；
- 查看版本历史；
- 激活历史版本。

要求：
1. 原始 JSON 高度受限，可滚动；
2. JSON 错误提示清晰；
3. 保存 DSL 时调用后端创建 TestCaseVersion；
4. 当前版本号显示明显；
5. navigate_path 步骤显示为“菜单路径导航”；
6. business_goal 显示为可读业务目标。

============================================================
十三、测试运行页调整
============================================================

测试运行页不再要求每次手工输入完整测试目标。

字段顺序：
1. 选择项目
2. 选择功能测试用例
3. 自动展示：
   - base_url
   - login_url
   - 测试账号
   - 当前 DSL 版本
   - 测试目标
   - 测试数据
4. 允许临时修改：
   - 测试数据
   - 视觉兜底
   - 本次运行设置
5. 操作按钮：
   - 执行用例
   - 保存本次修改到用例
   - 另存为新用例

要求：
1. 选择项目后加载用例。
2. 选择用例后自动带入 DSL 和账号。
3. 没有用例时提示去新增功能测试用例。
4. 仍保留“临时测试目标”模式，但要支持另存为用例。

============================================================
十四、验收标准
============================================================

请确保：

1. 可以在前端创建项目。
2. 可以配置项目 URL。
3. 可以配置项目测试账号。
4. 可以创建功能测试用例。
5. 可以编辑功能测试用例。
6. 可以保存 DSL。
7. 可以查看 DSL 版本。
8. 可以执行某个已保存用例。
9. 运行记录能在用例详情中查看。
10. 用例详情能查看失败分析和修复历史。
11. 测试运行页不再要求每次输入完整测试目标。
12. 前端 build 通过。
13. check.ps1 通过。
