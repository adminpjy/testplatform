# Prompt02：表格与查询自适应引擎

## 目标

解决企业系统中列表页差异大、查询条件隐藏、表格行打开方式不同、分页循环等问题。

## 能力

1. 自动识别查询入口：
   - 查询按钮。
   - 高级查询。
   - 筛选。
   - 展开。
   - 更多条件。

2. 自动识别查询字段：
   - label。
   - placeholder。
   - 表头。
   - 邻近文本。
   - 规则别名。

3. 自动识别表格：
   - table。
   - div grid。
   - ant table。
   - element table。
   - 自定义列表。

4. 自动处理行打开：
   - 点击第一列。
   - 点击标题列。
   - 点击操作列。
   - 双击行。
   - 新页面打开。
   - 弹窗打开。

5. 循环策略：
   - 当前页所有行。
   - 查询结果所有行。
   - 限制最大行数。
   - 失败行跳过或停止。
   - 每行处理后返回列表。

## LLM 使用

当规则不足时，向 LLM 提供：

1. 表头。
2. 行文本。
3. 按钮列表。
4. 页面截图。
5. 目标意图。

要求 LLM 输出可执行候选，不允许直接执行高风险动作。

## 验收标准

1. “读取待办列表，逐一点开待办”能生成稳定行循环。
2. 打开新页面后能完成处理并回到列表。
3. 过程截图能记录每一行的打开、处理、返回。
