# Prompt04：审批流程自适应引擎

## 目标

解决审批类系统中意见框、下一处理人、提交按钮、确认框和新页面差异问题。

## 识别内容

1. 审批意见区域：
   - 我的意见。
   - 审批意见。
   - 处理意见。
   - 意见。

2. 意见写入方式：
   - 追加。
   - 覆盖。

3. 下一处理人：
   - 保留已有默认值。
   - 选择处理人。
   - 不允许自动乱选。

4. 审批动作：
   - 提交。
   - 审批。
   - 同意。
   - 通过。
   - 办理。
   - 驳回。
   - 传阅。

5. 成功证据：
   - 成功提示。
   - 页面关闭。
   - 返回列表。
   - 待办数量变化。
   - 状态变化。

## 风险控制

1. 审批是高风险动作。
2. 必须确认目标单据明确。
3. 必须确认意见已填写。
4. 必须确认提交按钮匹配审批语义。
5. 未确认成功证据时，不得标记成功。

## 验收标准

1. “逐一点开待办，填写意见按要求执行，点击提交”能稳定执行。
2. 新页面打开审批单时能切换上下文。
3. 每个待办都有过程截图和结果。
