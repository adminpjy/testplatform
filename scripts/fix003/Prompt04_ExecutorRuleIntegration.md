# Prompt04：执行器接入规则中台

## 目标

让执行器真正使用规则，而不是只依赖硬编码、自然语言解析或 LLM。

## 接入位置

至少接入以下模块：

1. 登录目标执行器。
2. 菜单路径导航。
3. 元素定位器。
4. 表格处理器。
5. 表单填写处理器。
6. 审批处理器。
7. 断言处理器。
8. 恢复处理器。

## 执行优先级

动作执行时按以下顺序：

1. DSL 明确 selector 或 locator。
2. 用例级规则。
3. 页面级规则。
4. 项目级规则。
5. 系统级规则。
6. 企业级规则。
7. 通用规则。
8. LLM 候选判断。
9. 视觉兜底。
10. 人工介入。

## 用户可理解错误

如果规则缺失或规则失效，不要抛出低级异常。  
错误提示必须包含：

1. 当前想做什么。
2. 规则是否命中。
3. 页面上看到了什么。
4. 为什么没有继续。
5. 建议补充什么规则。
6. 是否可以一键生成规则草案。

## 验收标准

1. 登录按钮为 `Login`、`登录`、`Sign in` 时均可由规则处理。
2. 菜单选中但页面未跳转时，能通过规则判断“已进入目标区域”。
3. 审批按钮为 `提交`、`同意`、`通过`、`办理` 时能按规则识别。
4. 运行记录中能看到规则命中详情。
