# Prompt06：前端规则中心

## 目标

让测试人员和维护人员能低成本查看、审核、启用、停用、验证规则。

## 页面

在能力中心或新增规则中心中提供：

1. 规则列表。
2. 草案列表。
3. 规则详情。
4. 版本历史。
5. 规则 dry-run。
6. 规则启用/停用/回滚。
7. 规则命中记录。

## 交互要求

1. 不要求用户理解底层 JSON 才能操作。
2. 重要字段有表单化编辑。
3. JSON 高级编辑保留。
4. 高风险规则展示醒目标识。
5. 启用前展示影响范围。

## 验收标准

1. 能从失败分析跳转到规则草案。
2. 能一键创建规则并 dry-run。
3. 能启用规则后重跑失败用例。
