# Prompt02：规则生命周期与审核流程

## 目标

实现规则从“草案”到“可执行规则”的完整生命周期。

## 生命周期

规则状态必须至少包括：

1. `draft`
2. `pending_review`
3. `approved`
4. `active`
5. `disabled`
6. `rejected`
7. `archived`

## 操作

实现后端 API：

1. 创建规则草案。
2. 从 `RuleDraft` 转成 `AbilityRule`。
3. 编辑规则。
4. 提交审核。
5. 审核通过。
6. 拒绝。
7. 启用。
8. 停用。
9. 复制新版本。
10. 回滚到历史版本。

## 安全要求

1. 高风险规则必须人工确认。
2. 涉及审批、删除、作废、发布的规则，默认 `production_enabled=false`。
3. 规则启用必须记录操作者、原因、时间。
4. 不允许直接覆盖旧版本，必须产生新版本。

## 前端要求

1. 能查看规则草案。
2. 能查看正式规则。
3. 能对比版本。
4. 能一键从草案创建规则。
5. 能启用、停用、回滚。
6. 高风险规则需要明显提示。

## 验收标准

1. 失败分析生成的规则草案能进入审核队列。
2. 审核通过后能启用。
3. 启用后执行器可读取。
4. 回滚后执行器读取到回滚版本。
